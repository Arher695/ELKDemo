
# Необходимо развернуть ELK-стек с помощью Docker Compose и настроить запись и отображение логов из Spring Boot приложения.

## Инструкция к выполнению
Разработать Docker Compose файл для разворачивания ELK-стека. Файл должен содержать описание необходимых сервисов (ElasticSearch, Logstash, Kibana, Filbeat).
Создать SpringBoot-приложение.
Добавить logstash-logback-encoder стартер в приложение.
Настроить логирование с помощью filebeatAppender в конфигурационном файле логгера logback-spring.xml
Запустить приложение.
Выполнить целевое действие в приложении для записи логов в файл.
Проверить запись логов в файл.
Добавить конфигурационный файл для Filebeat.
При необходимости добавить конфигурационный файл для Logstash.
Запустить сервисы из Docker Compose файла.
Открыть web-интерфейс Kibana.
Перейти на вкладку Discover, создать index pattern, настроить поля для сортировки логов в интерфейсе по умолчанию.
Проверить отображение логов для созданного индекса во вкладке Discover.
Kibana по умолчанию хранит сохранённые дашборды, визуализации и настройки в индексе ElasticSearch (по умолчанию .kibana). Для того, чтобы Ваши настройки сохранились и были доступны ревьюверу, убедитесь, что в Docker Compose используется volume для сервиса ElasticSearch и ElasticSearch сохраняет данные в указанный volume. Данные из этого volume должны быть выложены вместе с репозиторием на GitHub. В противном случае ревьювер не сможет проверить корректность настройки Kibana.

### Чек-лист самопроверки результатов
Логи записываются в файл.
Индекс с логами появляется в ElasticSearch, доступен в Kibana для настройки index pattern.
После настройки index pattern логи успешно отображаются в Kibana.
После завершения всех настроек новые логи отображаются в Kibana автоматически.
